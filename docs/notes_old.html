noticed that 'autosize' causes a flicker when first loading an image. The image shows full size for a split second before autosize takes effect. This is because the resize code is in the onload.

I moved all the 'image_load' code into 'showFile' including the statusbar update. Directly calculating the image size before display using a temp image eliminates the flicker. Moving statusbar update into showfile reduces number of temporary 'images'. Removing the 'autosize' from 'image_load' saves us from the recursive 'infinite loop' problem that was originally overcome with flags.
